# 배포 메모

## 현재 출시 범위

이번 MVP는 양력 입력만 지원합니다.

음력 변환은 원본 개발본에 남아 있지만, 서버 환경에서 안정적으로 운영하기 전까지 출시본에서는 제외합니다.

## 권장 호스팅

현재 서비스는 정적 페이지가 아니라 Python 서버가 필요합니다.

1차 권장 후보는 Render입니다.

이유는 다음과 같습니다.

- GitHub 저장소 연결이 단순합니다.
- Python Web Service를 바로 실행할 수 있습니다.
- `render.yaml`로 배포 설정을 저장소 안에 둘 수 있습니다.
- 기본 `onrender.com` 주소와 HTTPS를 제공합니다.

대안 후보:

- Render
- Railway
- Fly.io

GitHub Pages는 이번 MVP에 맞지 않습니다.

## 실행 명령

```bash
python -m saju_web.app
```

호스팅 서비스가 `PORT` 환경변수를 제공하면 서버가 자동으로 해당 포트를 사용합니다.

## 배포 후 확인 주소

배포가 끝나면 아래 두 가지를 확인합니다.

```text
/
/api/judgment
```

첫 화면이 열리고, 프리미엄 분석 API가 정상 응답해야 합니다.

## Render 배포 값

Render에서 직접 입력해야 할 경우 아래 값을 사용합니다.

```text
Service Type: Web Service
Repository: https://github.com/nicehodong-tech/release_solar_mvp.git
Branch: main
Runtime: Python
Build Command: python -m compileall saju_web saju_analysis_engine saju_birth_engine
Start Command: python -m saju_web.app
```

## 도메인 연결

Render 서비스가 Live 상태가 된 뒤 진행합니다.

1. Render 서비스의 Settings로 들어갑니다.
2. Custom Domains에서 도메인을 추가합니다.
3. 도메인 구매처 또는 DNS 관리 화면에서 Render가 안내하는 DNS 값을 입력합니다.
4. Render에서 Verify를 누릅니다.
5. HTTPS 인증서가 발급된 뒤 실제 도메인으로 접속합니다.

## 쿠팡 파트너스 준비

이번 서비스의 1차 수익화는 Google AdSense가 아니라 쿠팡 파트너스를 기준으로 합니다.

현재 단계에서는 쿠팡 파트너스 링크나 배너 코드를 넣지 않습니다.

이유는 다음과 같습니다.

- 아직 쿠팡 파트너스 최종 링크와 배너가 정해지지 않았습니다.
- 운세 결과 화면 안에 성급하게 광고를 넣으면 상품 집중도가 떨어집니다.
- 제휴 링크가 들어가는 위치에는 대가성 고지 문구를 함께 표시해야 합니다.

기본 적용 방식은 다음과 같습니다.

1. 사용자가 사주 분석을 실행합니다.
2. 분석 결과를 생성합니다.
3. 결과 화면으로 이동한 직후 쿠팡 방문 팝업을 띄웁니다.
4. 사용자가 `쿠팡 방문하고 결과 보기`를 누르면 새 탭으로 쿠팡을 열고, 원래 화면에서는 분석 결과를 볼 수 있게 합니다.

분석 결과 카드 안에 광고를 섞지는 않습니다.

팝업은 화면을 dim 처리하고 중앙에 크게 노출하는 전환형 팝업으로 설계합니다.

팝업 기본 원칙은 다음과 같습니다.

- 우상단 닫기 버튼과 별도 계속 보기 버튼은 두지 않습니다.
- 쿠팡 이동 버튼은 광고임을 숨기지 않습니다.
- 대가성 고지 문구를 팝업 안에 함께 표시합니다.
- 운세 결과와 상품 추천을 같은 판단처럼 보이게 하지 않습니다.
- 사주 분석을 다시 실행하면 팝업도 다시 노출합니다.

제휴 고지 기본 문구는 다음을 사용합니다.

```text
이 페이지는 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받을 수 있습니다.
```

쿠팡 파트너스 공식 가이드에 맞춰 실제 링크나 배너가 보이는 화면에는 제휴 고지 문구가 함께 보여야 합니다.
